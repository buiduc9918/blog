Các bước xây dựng web blog khóa học bằng Node.js, Express, MongoDB và Handlebars

1. Khởi tạo dự án
- Tạo thư mục dự án và chạy lệnh: npm init -y
- Cài các package cần thiết:
  - npm install express mongoose hbs cookie-parser method-override slugify
  - npm install --save-dev nodemon morgan express-handlebars sass prettier husky lint-staged
- Tạo file package.json với các script hữu ích:
  - npm start: chạy server bằng nodemon
  - npm run watch: theo dõi SCSS
  - npm run seed: chèn dữ liệu mẫu vào MongoDB

2. Tổ chức thư mục theo mô hình MVC
- src/index.js: điểm khởi chạy ứng dụng
- src/routes/: định nghĩa route cho các trang
- src/app/controllers/: xử lý logic cho từng route
- src/app/model/: định nghĩa schema và model MongoDB
- src/config/db/: kết nối database
- src/resource/views/: giao diện Handlebars
- src/public/: file tĩnh như CSS, ảnh, JS
- src/resource/scss/: file SCSS để viết style

3. Kết nối MongoDB
- Cài MongoDB và đảm bảo dịch vụ đang chạy
- Tạo kết nối trong src/config/db/index.js
- Dùng mongoose.connect() để kết nối tới database
- Nếu cần, dùng seed để tạo dữ liệu mẫu cho trang chủ và khóa học

4. Xây dựng route và controller
- Tạo route cho trang chủ, chi tiết khóa học, tạo khóa học, sửa, xóa
- Controller nhận request, truy vấn database, rồi render view bằng Handlebars
- Luôn kiểm tra đầu vào và xử lý lỗi một cách rõ ràng

5. Thiết kế giao diện
- Dùng Handlebars để chia layout, header, footer và các partials
- Dùng SCSS để quản lý style thay vì viết trực tiếp CSS dài dòng
- Chia nhỏ giao diện thành các component và template dễ bảo trì

6. Thêm dữ liệu và chức năng nâng cao
- Tạo model Course với các trường như name, description, image, videoID, slug
- Dùng slugify để tạo URL thân thiện
- Thêm chức năng xem chi tiết khóa học và hiển thị video bằng iframe

7. Kiểm tra và chạy ứng dụng
- Cài đặt dependencies: npm install
- Seed dữ liệu mẫu: node src/config/seed.js hoặc npm run seed
- Chạy app: npm start
- Mở trình duyệt tại: http://localhost:3000

8. Quy trình Git và commit
- Kiểm tra thay đổi: git status
- Thêm file vào staging: git add .
- Commit theo chuẩn rõ nghĩa:
  - feat: thêm tính năng mới
  - fix: sửa lỗi
  - chore: chỉnh sửa cấu hình hoặc tiện ích
  - style: chỉnh giao diện
  - docs: cập nhật tài liệu
- Ví dụ:
  - git commit -m "feat: add course detail page"
  - git commit -m "fix: resolve course render error"
  - git commit -m "docs: update project readme"
- Đẩy lên remote:
  - git push origin main

9. Gợi ý workflow làm việc hàng ngày
- Tạo nhánh mới cho từng task: git checkout -b feature/ten-tinh-nang
- Làm việc trên nhánh riêng, commit nhỏ và rõ ràng
- Trước khi push, kiểm tra lại bằng git status và git diff
- Nếu cần, merge nhánh vào main sau khi review

10. Lưu ý quan trọng
- Đảm bảo MongoDB đang chạy trước khi khởi động ứng dụng
- Nếu thiếu package thì cài lại bằng npm install <package>
- Luôn giữ code sạch, comment hợp lý và commit đúng mục đích
